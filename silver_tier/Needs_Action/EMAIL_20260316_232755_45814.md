razashariff left a comment (openai/openai-agents-python#924)



Session ID loss is a symptom of relying on transport-level session state for MCP. MCPS (MCP Secure) provides session-independent message identity — every JSON-RPC message carries an ECDSA P-256 signed envelope with passport_id, unique nonce, timestamp, and SHA-256 hash of the full message body.



This means message authenticity and agent identity verification work regardless of whether the session ID is preserved, the connection is dropped, or the transport changes mid-conversation. The signing is at the message layer, not the session layer.



Published as an [IETF Internet-Draft](https://datatracker.ietf.org/doc/draft-sharif-mcps-secure-mcp/) and available on npm as [`mcp-secure`](https://www.npmjs.com/package/mcp-secure) (zero dependencies).
