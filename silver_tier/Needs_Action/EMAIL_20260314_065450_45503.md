## Problem



When `nest_handoff_history=True` (the default), `to_input_list()` produces unparsable data that the API rejects with orphaned `function_call` / `function_call_output` items.



### Root Cause



During a handoff with nesting:

1. `self.input` is replaced with the nested summary (a single assistant message containing `&lt;CONVERSATION HISTORY&gt;` text that embeds the function_call items as text lines)

2. `self.new_items` retains the **unfiltered** session items (including raw `function_call` and `function_call_output` items from the pre-handoff agent)

3. `_model_input_items` is populated with the **filtered** items that are consistent with the nested `self.input`



`to_input_list()` was concatenating `self.input` (nested summary) + items from `self.new_items` (unfiltered), producing:

- The nested summary (which already contains function_call info as text)

- **Plus** orphaned structured `function_call` / `function_call_output` items without a parent assistant message



The API rejects these orphaned items.



## Fix



`to_input_list()` now prefers `_model_input_items` when populated (after handoffs) over `self.new_items`. This ensures the items are consistent with the (possibly nested) `self.input`. Falls back to `self.new_items` for non-handoff runs where `_model_input_items` is empty.



## Tests



Added 2 tests to `test_handoff_history_duplication.py`:

- `test_to_input_list_uses_model_input_items_when_available` — verifies no orphaned function_call items after handoff

- `test_to_input_list_falls_back_to_new_items_without_handoff` — verifies normal non-handoff behavior unchanged



All 40 handoff/result tests pass.
