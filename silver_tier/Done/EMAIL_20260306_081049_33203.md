## Problem



The lock pattern in `SQLiteSession` creates a new `threading.Lock()` object for each operation when using file-based databases:



```python

with self._lock if self._is_memory_db else threading.Lock():

    # ...

```



This provides no thread safety for concurrent access to file databases, as each call uses a different lock instance.



## Solution



Use the shared `self._lock` for both memory and file databases:



```python

with self._lock:

    # ...

```
