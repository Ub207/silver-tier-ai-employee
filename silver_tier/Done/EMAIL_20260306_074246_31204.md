AbdulSamad94 left a comment (openai/openai-agents-python#1794)



> If `model_settings.parallel_tool_calls: False` works for your use case, we're open to adjust the example just by setting the option.



I tried with



```python

Agent(..., model_settings=ModelSettings(parallel_tool_calls=False),

```



but Gemini still fails:



* With `False`: `BadRequestError: Request contains an invalid argument`

* With `True`: `BadRequestError: Parallel tool calls are not supported`



So just setting the flag doesn’t fix the issue.

The root cause is that the example allows overlapping function calls when the model ignores the “one-at-a-time” instruction.

Maybe we should tighten the instructions (or add explicit multi-call handling) so it’s more robust across models like Gemini.
