### Body (preview)

This pull request adds a new example demonstrating an advanced deterministic workflow pattern using a state machine approach for guided, interactive agent flows.



## Overview



The new example deterministic_workflow.py showcases how to implement a travel booking system with a state machine pattern where:



- Tools are dynamically enabled/disabled based on the current step using is_enabled

- Agent instructions change dynamically per step for precise guidance

- A shared WorkflowState object tracks progress through 4 distinct steps

- The flow supports conversational interaction rather than rigid predefined questions



## Key Features



1. State Machine Pattern: Implements 4 clear steps:

   - Step 1: Collect origin and destination (save_locations tool)

   - Step 2: Check transport availability (check_transport_availability tool)

   - Step 3: Book chosen transport mode (book_transport tool)

   - Step 4: Finish and show summary (finish_workflow tool)

2. Dynamic Tool Management: Tools are conditionally enabled via is_enabled lambda functions that check the current step, restricting the agent to only relevant tools at each stage.

3. Adaptive Instructions: The agent&#39;s system prompt changes dynamically based on the current step, providing precise guidance for each phase of the workflow.

4. State Persistence: Uses a WorkflowState dataclass passed through RunContextWrapper to maintain context across turns.

5. Auto-mode Support: Includes compatibility with EXAMPLES_INTERACTIVE_MODE=auto for automated testing.

6. Error Handling: Provides graceful handling for mid-flow location updates via the update_trip_details tool.
