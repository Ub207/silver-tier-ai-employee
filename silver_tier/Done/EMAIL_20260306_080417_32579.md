**<sub><sub>![P1 Badge](https://img.shields.io/badge/P1-orange?style=flat)</sub></sub>  Preserve optional fields when serializing structured tool outputs**



When `ToolOutputImage` or `ToolOutputFileContent` is converted for the Responses API, the serializer only emits the `image_url` or `file_data` fields. If a tool instead returns `ToolOutputImage(file_id="…", detail="high")` or `ToolOutputFileContent(file_id="…", filename="report.pdf")`, those values are dropped and the produced payload becomes `{"type": "input_image", "image_url": None}` / `{"type": "input_file", "file_data": None}`. That misses the required identifier for the resource and causes the model call to fail for any output that relies on `file_id`, `file_url`, `detail`, or `filename`. The conversion should forward whichever optional attributes were provided so that non-text tool outputs remain valid.
