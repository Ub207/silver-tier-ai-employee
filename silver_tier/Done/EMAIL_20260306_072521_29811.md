**[P2] Persist modified session inputs when using callbacks**



The new `session_input_callback` is applied only when preparing the model input, but the conversation saved back to the session still uses the unmodified `input` value (`await self._save_result_to_session(session, input, result)`). If a callback rewrites or filters the user input/history (e.g. to replace existing history or truncate inputs) the model will see the modified list while the session will store the original values, so the next turn will reload stale messages and undo the callback’s intent. To keep session memory consistent with the callback’s output, `_save_result_to_session` should receive the already combined/filtered list or otherwise allow the callback to influence what is persisted.
