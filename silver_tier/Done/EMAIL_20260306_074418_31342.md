There's More Than Python in Your Environment











Written by Steve Croce (Field CTO), Anaconda Team









One of the greatest things about working with Python is the massive ecosystem 

of libraries that has been built around it. Need to work with image files? 

Install pillow, import it, and use it for easy image loading and manipulation. 

Need to plot some data? Install matplotlib, seaborne, or plotly, import it, and 

go. Need to build a machine learning model? Install scikit-learn, import it,... 

and you get the point. I know this is day 1 stuff, but if this is news to you, 

I regret to inform you that you’re writing way too much code… You should be 

building on the shoulders of giants. 







However, one thing that’s a little less known or understood is how much 

non-Python is in your Python environment when you install packages, like those 

above. You’ll likely find compiled binaries from C/C++, Go, Rust, Fortran, and 

more throughout your system. This may be surprising, but it’s what makes the 

Python ecosystem so rich. It not only enables developers to reuse algorithms 

written in those other languages within Python, but also provides a path to 

optimizing library functions for performance, flexibility, and compatibility 

with other tools. You get to benefit from all of that goodness, without having 

to leave the comfort of your Python code.
