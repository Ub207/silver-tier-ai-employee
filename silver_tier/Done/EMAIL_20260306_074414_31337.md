Thanks, @panilya. We came to the same conclusion about running LLMObs with the dd agent. 

Datadog patch adds a flag when patching to the agents sdk (agents._datadog_patch = True).

The reason datadog does not work afterwards is [this condition](https://github.com/DataDog/dd-trace-py/blob/602df92bf429815fcbd2bfb466d95a2aa3f1ef54/ddtrace/contrib/internal/openai_agents/patch.py#L56).



If you unpatch dd from agents sdk before `LLMObs.enable()`, tracing should work again.



```

set_trace_processors([])

# unpatch dd 

agents._datadog_patch = False

LLMObs.enable()

```
