Why I Stopped Typing My AI Prompts

**********************************



I type around 70 words per minute. Not bad. But I speak over 200

words per minute. That’s a 3× speed difference.



For years, I never thought much about it - typing was “good

enough.” But when I started using AI daily, that gap became

impossible to ignore.



Here’s why: AI outputs are only as good as the context you give

them. The more detailed your instructions, the better the

results. Yet typing out those details - edge cases, requirements,

expected outputs - slowed me down so much that I would cut

corners. I’d type half-prompts, settle for mediocre answers, and

fix the rest myself.



Then I started using Wispr Flow who I’m happy to say is not a

sponsor of my channel and this newsletter! Flow is a dictation

app for iOS and desktop that delivers fast, accurate

voice-to-text across any app you use. It’s context-aware, meaning

it adapts how your words are transcribed depending on where

you’re typing, and it syncs your custom dictionary and notes

between devices. The free plan lets you try up to 2,000 words a

week, making it easy to test before upgrading.



Speaking Full Context vs. Typing Half Prompts

---------------------------------------------



The first time I tried Flow, I just spoke my thought process:



“Write a Python function that parses JSON, but throw an exception

if any of the keys don’t match this schema. It also needs to

handle null values gracefully, log a warning if any optional

fields are missing, and return a default dictionary if parsing

fails.”



Flow instantly turned that ramble into a clean, structured

prompt. I pasted it into my AI tool and got an answer I could

actually use right away.



Typing that same instruction would’ve taken me ~2 minutes?

Speaking it? 20 seconds.
