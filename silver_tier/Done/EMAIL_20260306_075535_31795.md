### Describe the feature



#### Problem Statement



I'm leveraging [lifecycle events](https://openai.github.io/openai-agents-python/ref/lifecycle/) to record timing metrics for my AI agent. I have a class that inherits `RunHooksBase`. When [on_agent_start](https://openai.github.io/openai-agents-python/ref/lifecycle/#agents.lifecycle.RunHooksBase.on_agent_start) and [on_tool_start](https://openai.github.io/openai-agents-python/ref/lifecycle/#agents.lifecycle.RunHooksBase.on_tool_start) are called, I begin timers. When [on_agent_end](https://openai.github.io/openai-agents-python/ref/lifecycle/#agents.lifecycle.RunHooksBase.on_agent_end) or [on_tool_end](https://openai.github.io/openai-agents-python/ref/lifecycle/#agents.lifecycle.RunHooksBase.on_tool_end) are called, I stop timers and record the metric. Unfortunately, I'm unable to differentiate between tool calls. Thus, metrics may be slightly imprecise if the agent has parallel tool calls enabled. Knowing with `*_start` corresponds with which `*_end` would allow this type of observability.



#### Feature Details



If possible, all hook methods should have access to a unique ID that identifies the tool call. I notice that elsewhere, [ToolContext](https://github.com/openai/openai-agents-python/blob/f3cac173055e180752167d0327a241240a2248a2/src/agents/tool_context.py#L22) provides a `tool_call_id`. This type of value would be ideal for lifecycle events and satisfy the problem statement above.
