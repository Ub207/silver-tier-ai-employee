NateRVance left a comment (openai/openai-agents-python#1605)



> It seems your `fc_agent` agent, which as tools, runs with a different model. The agent with with gpt-oss does not have tools.



This is correct. Here's the still-open feature request to use gpt-oss for tool calling with the chat completions API in vllm: https://github.com/vllm-project/vllm/issues/22879. My understanding is that this support has been implemented but not yet released.



The bug that I am reporting is with passing tool call outputs generated using some other model (qwen in my example code) to gpt-oss as input. From what I can tell, there is an incompatibility between openai-agents tools and harmony, which aught to be reconciled in one of three ways:



- openai-agents could add a name field to support harmony's format.

- vLLM could generate a dummy name field if it isn't provided by the caller (I don't know what affect this might have on performance).

- harmony could gracefully handle this situation.



As far as I can tell, openai-agents has the tool name internally and could provide it, potentially boosting performance should harmony benefit from it, so this bug should be fixed here. But if supporting harmony-based models is out of scope for this project, then should I report this incompatibility elsewhere?
