Yes, the OpenAI Agents SDK supports connecting to multiple MCP servers from a single agent, including servers using the stdio transport.



The Agent class constructor accepts a list of MCPServer instances in its mcp_servers parameter. You can instantiate multiple MCPServerStdio objects, each configured for a different local process, and pass them all in that list. The agent will then have access to the tools exposed by all of the connected servers.



How It Works

Based on the documentation and source code, the Agent is designed to handle multiple MCP servers.



In [c:\Users\vp431\OneDrive\Desktop\openai-agents-main\src\agents\agent.py](code-assist-path:c:\Users\vp431\OneDrive\Desktop\openai-agents-main\src\agents\agent.py), the AgentBase class defines mcp_servers as a list:



python

mcp_servers: list[MCPServer] = field(default_factory=list)

The agent then fetches tools from all servers in this list. This applies to any MCPServer implementation, including MCPServerStdio.
