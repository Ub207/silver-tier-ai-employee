@sridharvenkat @amirna2 I looked into this issue and my best guess is that:



1. In one of your previous function calls, the output didn’t include `image_url`. You might have accidentally returned `None`, for example:



   ```python

   @function_tool

   def fetch_image() -> ToolOutputImage | ToolOutputImageDict:

       return {

           "type": "image",

           "image_url": None,

       }

   ```



   When this gets sent to the OpenAI API, it triggers exactly the error you mentioned:



   ```

   openai.BadRequestError: Error code: 400 - {'error': {'message': "Missing mutually exclusive parameters: 'input[2].output[0]'. Ensure you are providing exactly one of: 'file_id' or 'image_url'.", 'type': 'invalid_request_error', 'param': 'input[2].output[0]', 'code': 'missing_mutually_exclusive_parameters'}}

   ```



2. The bigger issue is that the current `SQLiteSession` implementation stores that problematic function output.

   Once this happens, the conversation thread becomes permanently broken: every subsequent request will hit the same error, because that bad function call output will always sent back to the API.



A temporary workaround is to start a new thread (i.e., use a different `session_id`). The root fix might be to patch the Session mechanism — it shouldn’t store items that cause API errors when a bad response occurs.
