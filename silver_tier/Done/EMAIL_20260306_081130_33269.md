**<sub><sub>![P1 Badge](https://img.shields.io/badge/P1-orange?style=flat)</sub></sub>  Shared lock never used for file-based SQLite sessions**



The new `_lock_context()` still only returns `self._lock` when `_is_memory_db` is true, while file-backed databases now use `nullcontext()`, i.e. no locking at all. This leaves concurrent access to file-based sessions completely unsynchronized, so the race condition described in the commit message persists. To actually serialize access you need to return the same `self._lock` for file-backed databases as well (or just use `with self._lock:` directly).
