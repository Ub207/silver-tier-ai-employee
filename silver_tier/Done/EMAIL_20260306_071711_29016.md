A trick that I did is to configure the agent's `tool_use_behavior` to use `StopAtTools` to stop execution at the tool function that corresponds with requesting human input. In this way, you can get out of the agent's execution loop (in which the `SQLiteSession` will automatically persist the session in the DB), then request input from human (either through direct stdio input or using other external system), and then later on resume session again.



Example code:



```

class HumanInputRequired(BaseModel):

    question: str





@function_tool

async def get_human_response(

    question: str,

) -> HumanInputRequired:

    """This is a tool to get answer from human. Please use this tool anytime you require input or feedback from human.



    Args:

        question (str): The question to be asked to the human."""

    return HumanInputRequired(question=question)





agent = Agent(

    name="..."

    ...

    tools=[get_human_response],

    ...

    tool_use_behavior=StopAtTools(stop_at_tool_names=[get_human_response.name]),

)



session_id = str(uuid.uuid1())



session = SQLiteSession(session_id, "agent_sessions.db")



response = await Runner.run(agent, query, session=session)

```



Then some options of how to handle the response.

- Option 1: simple looping:



```

while isinstance(response.final_output, HumanInputRequired):

    human_response = input(f"Human input required! Question: {response.final_output.question}\nPlease answer here: ")

    response = await Runner.run(agent, human_response, session=session)

```



- Option 2: if the human input will come at much later time:



```

if isinstance(response.final_output, HumanInputRequired):

    # Here you might be requesting human input from external system.

    # The sesssion ID is needed here for future reference to match the human input with the right session.
