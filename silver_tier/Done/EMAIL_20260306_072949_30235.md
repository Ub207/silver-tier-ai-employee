I can consistently reproduce the 400 error when running an agent with GPT-5 (Responses API) that uses Code Interpreter to read a PDF and then return structured JSON. The first turn emits a reasoning item and one or more code_interpreter_calls. On the next turn, the SDK appears to resend the prior reasoning item without a matching follower, and the Responses API rejects it with:

```

BadRequestError: 400 - {

  "error": {

    "message": "Item 'rs_xxx' of type 'reasoning' was provided without its required following item.",

    "type": "invalid_request_error",

    "param": "input"

  }

}

```

- openai 1.107.1



- openai-agents 0.2.11



- Backend: Azure OpenAI (AsyncAzureOpenAI)



- Model/deployment: gpt-5-… (reasoning enabled)



- OS: Windows 11 / Python 3.11 (but repro isn't OS-specific)

1. Upload a PDF to the agent’s Code Interpreter container.

3. Prompt the agent to extract structured fields and return only JSON (validated via Pydantic).
