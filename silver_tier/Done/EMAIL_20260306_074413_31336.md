@dhruvrnaik you can use this ENV var https://ddtrace.readthedocs.io/en/stable/configuration.html#DD_PATCH_MODULES to enable LLMObs modules you heed.



`set_trace_processors([])` would work only if you're running datadog in agentless mode, if you're running datadog agent - `set_trace_processors([])` won't work (at least it didn't work for me).



If you're using datadog agent, you wouldn't be able to remove default tracer because datadog agent setup ups `openai-agents` custom tracing provider during app start, If I'm not wrong.



As a workaround, I've added a logs filter to check 403 Zero-Data retention policy error messages and filter them, which helped me not to spam my logs.



@seratch What do you think about an idea to add a new env variable to be able to disable default library's tracing, while keeping tracing enabled?



I can see there is a `OPENAI_AGENTS_DISABLE_TRACING` env var, which completely disables tracing and would be great to have additional env var to only disable default tracing provider.
