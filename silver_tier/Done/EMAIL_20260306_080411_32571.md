@tnduyh5 I believe there’s a misunderstanding, and both of these are not bugs:



1. FunctionTool and Handoff are different concepts.



Handoff is not an actual tool, it doesn’t need to have the same attributes as a FunctionTool.

The all_tools list in the Agent does not include handoffs, because handoffs are handled separately and only merged into the final tools parameter when making the actual API call.



2. Handoff is supported with the ChatCompletions API.



It runs fine in practice, and you’re confusing hosted tools with handoffs.

The handoff logic is not processed inside tool_to_openai; it’s handled by convert_handoff_tool, which is also called in litellm_model.py.



Anyway, the SDK already handles this correctly, I believe there’s no need to fix it.
