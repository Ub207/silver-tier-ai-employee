@muneebarshadd There’s no bug here, and you’re just confusing two different kinds of hooks. 



There are two kinds of hooks:



* AgentHooks: attached to each agent (Agent(hooks=...))

* RunHooks: attached to the runner (Runner.run(..., hooks=...))



You can check these examples to see how it works:



* https://github.com/openai/openai-agents-python/blob/main/examples/basic/lifecycle_example.py

* https://github.com/openai/openai-agents-python/blob/main/examples/basic/agent_lifecycle_example.py



Here’s your example fixed and working correctly 👇



```

from typing import Optional

from agents.items import TResponseInputItem



class MyAgentHooks(RunHooks):

    def __init__(self):

        self.event_counter = 0



    async def on_agent_start(self, context: RunContextWrapper, agent: Agent) -> None:

        print("✅ on_agent_start called")  # This IS called

    

    async def on_llm_start(

        self,

        context: RunContextWrapper,

        agent: Agent,

        system_prompt: Optional[str],

        input_items: list[TResponseInputItem],

    ) -> None:

        print("✅ on_llm_start called") # This IS called

        print(f"Response: {response}")

        print("✅ on_handoff called")  # This IS called
