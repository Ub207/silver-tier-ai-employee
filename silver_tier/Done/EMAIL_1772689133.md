## Summary

- keep successful function tool outputs when a sibling parallel tool raises or is cancelled

- convert per-tool failures into normal tool output items instead of aborting the whole turn

- add a regression test covering a cancelled sibling tool



## Why

A SEVBot investigation issue exposed that parallel function tool execution is currently all-or-nothing: if one sibling tool raises, the SDK can fail to materialize any parent tool outputs for that turn, which can leave the caller with no final answer. This change isolates failures per tool so completed siblings still produce outputs and the agent can continue.
