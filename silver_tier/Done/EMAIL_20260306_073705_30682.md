I am also seeing this issue.



When an Input Guardrail triggers (e.g. math_homework_guardrail), the Agent execution is halted for the user, but in the traces I can still see that the main Agent ("Customer Support") generated an answer. This causes unexpected token usage and cost implications.



Steps to reproduce (simplified):

1. Define an input guardrail Agent (Gemini model).

2. Run a main Agent with that guardrail.

3. Send input like "Hello!? 2 + 2 = ?"

4. Guardrail tripwire is correctly raised (exception in terminal).

5. But traces show the Customer Support Agent generated an answer, and tokens were consumed.



Expected:

- When guardrail tripwire is triggered, main Agent generation should be skipped entirely (no cost, no traces).



Actual:

- Guardrail triggered, but main Agent still generated an answer in traces, and tokens were billed.



Environment:

- SDK version: 0.2.3
