hussain9491 created an issue (openai/openai-agents-python#1618)





In the current SDK, the Agent class defines the name parameter as type str. However, the constructor also accepts other types (e.g., int) without raising any error.



This creates a mismatch between the type annotation and the actual behavior.



Current Behavior

from openai import Agent  



# This works even though type hint says str

agent = Agent(name=123)  

print(agent.name)  # Outputs: 123



Expected Behavior



The name parameter should be consistent with its type annotation. There are two possible solutions:



Option A (Recommended): Strict enforcement



Keep name: str but raise an exception if a non-string value is provided.



if not isinstance(name, str):

    raise TypeError(f"Agent.name must be str, got {type(name).__name__}")





Option B (Flexible typing)



Update the type annotation to reflect reality, for example:



name: Union[str, int]  





or



name: Any



Why This Matters



Aligns the SDK’s type annotations with its actual behavior.



Prevents confusion for developers using type checkers (mypy, IDEs, etc.).



Improves type safety and consistency in the SDK
