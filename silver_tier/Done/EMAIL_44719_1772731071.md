The MCP spec defines two distinct human-readable fields on a Tool object: title (and annotations.title) as a short display label, and description as a long-form explanation intended for the model. These serve different audiences and should be kept separate. The SDK currently conflates or drops both in a few places.



ToolCallItem has no title field. PR #2381 added ToolCallItem.description to expose tool metadata to streaming UIs and observability tooling, but there is no corresponding title field. A consumer that wants to display a short human-readable label alongside a full description has no way to get that from the item  description is the only option, and it maps to the wrong thing.
