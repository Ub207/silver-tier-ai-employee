# OpenAI Agents SDK v0.3.3 - Handoff Bugs Fix



## Overview



Fixed **TWO CRITICAL BUGS** in `openai-agents` SDK v0.3.3 that prevented Handoff from working:



1. ❌ **Bug #1**: `AttributeError: 'Handoff' object has no attribute 'name'`

2. ❌ **Bug #2**: `UserError: Hosted tools are not supported with the ChatCompletions API`



Both bugs are in the **upstream SDK**, not our code!



---



## Bug #1: Missing `name` Attribute



### Issue



```

AttributeError: 'Handoff' object has no attribute 'name'

```



Occurred in `agents/run.py` line 903:



```python

tool_names = [t.name for t in all_tools]

```



### Root Cause



Inconsistency in the SDK:



1. `Handoff` class has `tool_name` attribute ✅

2. `FunctionTool` class has `name` attribute ✅

3. SDK's `run.py` assumes ALL tools have `.name` ❌

4. Result: `AttributeError` when accessing `Handoff.name`



### Solution for Bug #1



Add a `name` property to `Handoff` class:



```python

@property

def name(self):

    return self.tool_name



Handoff.name = name

```
