## Pull Request Description



### Summary



This pull request introduces **Workflow System v1 (Beta)** - a declarative orchestration engine for complex multi-agent interactions. This beta release provides the foundational framework for defining sophisticated agent coordination patterns through a clean, type-safe API that supports conditional execution, multiple connection types, and advanced routing logic. As a v1 beta feature, this implementation establishes the core architecture while allowing for future refinements based on user feedback and real-world usage patterns.



### Test plan



The beta implementation includes:

- **Unit tests** for all new workflow classes and connection types

- **Integration tests** demonstrating workflow execution patterns

- **Example applications** showcasing real-world use cases

- **Documentation** with comprehensive API references and usage guides

- **Type safety validation** ensuring proper generic type constraints



All tests pass and the examples have been validated to work correctly with both synchronous and asynchronous execution modes. The beta status allows for iterative improvements based on community feedback.



### Issue number



This PR introduces a new beta feature and does not address a specific issue number.



### New Classes and Objects



**Core Workflow Classes (Beta v1):**

- `Workflow[TContext]`: Main workflow orchestrator with validation, execution, and tracing capabilities

- `WorkflowResult[TContext]`: Encapsulates workflow execution results including step results and skipped connections

- `Connection[TContext]`: Abstract base class for all agent connection types



**Connection Types (Beta v1):**

- `HandoffConnection[TContext]`: Transfers control between agents via handoff mechanism

- `ToolConnection[TContext]`: Uses target agent as a function tool of the source agent

- `SequentialConnection[TContext]`: Passes output from one agent as input to another

- `ConditionalConnection[TContext]`: Route
