Tool call failures in Continue often come from model capability mismatches -- not all models support function calling, and the ones that do have different schemas for tool definitions.



The routing issue: if Continue is sending tool-formatted requests to a model that does not support them, you get silent failures or malformed responses instead of a useful error.



A routing layer can solve this by ensuring tool-call requests only go to models with confirmed tool support. Komilion routes tool-heavy requests to capable models automatically, falling back gracefully if needed. komilion.com if useful.
