**Summary**

Add a first-class, metric-driven optimization loop to the Agents SDK—akin to [[DSPy’s optimization workflow](https://dspy.ai/learn/optimization/overview/)](https://dspy.ai/learn/optimization/overview/)—so users can automatically improve prompts, instructions, tools/graph choices, and agent parameters against an evaluation set.



---



## Motivation



* Most agent builders already maintain an eval set (unit tests, golden Q/A, task fixtures).

* Today, closing the loop (→ run eval → tweak prompts/tools → re-eval) is manual and brittle.

* A built-in `optimize` primitive would make agents **measurably better**, **faster**, and **reproducible** without forcing users into external tooling.



Concrete wins:



* **Performance**: metric-guided search (EM/F1/accuracy, custom judges) over prompts, system messages, tool selection, temperatures, and chain parameters.

* **Reproducibility**: tracked trials, seeds, and artifacts.

* **DX**: one API/CLI to run, compare, and export the best configuration back into the agent.



---



## Prior Art



* **DSPy**: declarative modules “compiled” into optimized pipelines via dataset- and metric-driven search.

* **MLflow GenAI Optimize**, **promptfoo**, **Llama Prompt Ops**: each covers slices of this problem but outside the Agents SDK.



---



## Proposal



Introduce an **Optimize API** that runs iterative search over agent configs against a dataset + metric, with pluggable strategies (e.g., evolutionary/mutation, MIPRO-style, Bayesian search, grid/random, CEM).
