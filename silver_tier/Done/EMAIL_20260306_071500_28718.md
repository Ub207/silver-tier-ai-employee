###  Summary

This PR fixes an issue when using **Gemini with tools and structured outputs** in LiteLLM.  

Previously, enabling structured outputs with tools caused a `400 error` response from Gemini.  



### Fix

- Added an **optional text/plain fallback** (via env flag).  

- If structured outputs are not supported, the request now falls back to `text/plain` instead of failing.  

- Default behavior is unchanged, so this update is backward-compatible.  



### Why

- Prevents Gemini + tools workflows from breaking.  

- Makes the developer experience smoother.  

- Aligns Gemini behavior with how agents already handle structured outputs.  



### Testing

- Verified locally with Gemini API keys.  

- **Agents + structured outputs → works **  

- **Tools + structured outputs → now gracefully falls back **
