## Problem



Currently, handoffs in the Agents SDK rely on LLM tool-calls, which leads to:



- **Lower determinism** – LLM may choose the wrong agent.

- **Inefficient routing** – extra reasoning step required.

- **Poor performance** – smaller models with weak tool-call capabilities struggle.



## Proposal



Add a **deterministic handoff mechanism** that allows developers to:



1. **Explicitly specify the target agent** for handoff, bypassing LLM decision-making.

2. **It could be possible to define a function** that uses comtext data to decide whether to hand off, or more specifically, which agent to hand off to.



## Benefits



- **Higher determinism & reliability**  

- **Better performance** – skip extra LLM reasoning step  

- **Enables graph-based agent orchestration** (DAG workflows)  

- **Improves compatibility with small models**
