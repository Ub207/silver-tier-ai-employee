vrtnis left a comment (openai/openai-agents-python#1057)



> I have a question regarding the nested stream. Only the tool events seem to be passed to the parent. What is that?



short answer: yes, by design the PR bubbles up tool_called and tool_output from nested runs. the goal was to expose “what tools ran and what they returned” without flooding the parent stream with every inner event (model tokens, traces, etc.) or breaking existing uis. concretely, inside Agent.as_tool(..., stream_inner_events=True) we forward only those two names from the child RunItemStreamEvents into the parent’s queue. that keeps ordering sane and makes it easy for callers to render nested tool activity (avoids potentially surprising interleaving and ui regressions).
