### Question

I am trying to build an agent using a local model (meta-llama/Llama-3.1-8B-Instruct). The workflow essentially looks like this: Openai-agents - > liteLLM -> ollama (see code below). The agent keeps calling the tool over and over again without producing an output. However, when I point LiteLLM to a Claude API, the issue disappears. IS this an issue or user error? (see code snippet below  - logs also attached)



## Llama implementation



from __future__ import annotations

from pydantic import BaseModel

import asyncio

from agents import Agent, Runner, function_tool, set_tracing_disabled

from agents.extensions.models.litellm_model import LitellmModel



set_tracing_disabled(disabled=True)



[debug.txt](https://github.com/user-attachments/files/21911233/debug.txt)



@function_tool

def get_weather(city: str):

    print(f"[debug] getting weather for {city}")

    return f"The weather in {city} is sunny."



async def main(model: str, api_key: str):



    agent = Agent(

        name="Assistant",

        instructions="You are a helpful assistant",

        model=LitellmModel(model=model, api_key=api_key),

        tools=[get_weather],



    )



    result = await Runner.run(agent, 'What is the weather in Tokyo', max_turns=10 )

    print(result.final_output)





# Configuration



model = 'ollama/llama3:latest'

api_key = ''



litellm._turn_on_debug()

# For Jupyter - just await directly:

await main(model, api_key)
