I noticed that the Agent class defines name as type str, but in practice it also accepts other types (e.g., int) without raising an error.



To make this consistent, I suggest either:



Keep name: str but raise an exception if a non-string is passed,



if not isinstance(name, str):

    raise TypeError(f"Agent.name must be str, got {type(name).__name__}")





Or update the annotation to Union[str, int] (or Any) so that the type hint matches actual behavior.



This will make the SDK type hints accurate and prevent confusion for developers using type checkers.
