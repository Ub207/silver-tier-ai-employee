@AbdulSamad94 



The root cause is not that this example "allows" parallel function calls. OpenAI models natively support parallel function calling, so there’s no need to artificially disable it.



Based on your error, I assume you’re directly using Gemini’s OpenAI-compatible endpoint.



The real issue is with Gemini’s OpenAI-compatible endpoint implementation. See related discussions:



* https://discuss.ai.google.dev/t/gemini-openai-compatibility-issue-with-tool-call-streaming/59886

* https://discuss.ai.google.dev/t/tool-calling-with-openai-api-not-working/60140



These are incompatibilities on the Gemini side, not something the SDK example itself needs to workaround. 

I’d suggest reporting them to Google so they can fix their OpenAI endpoint behavior.



I also tested LiteLLM support. Unfortunately, it runs into other errors — in this case a `json.decoder.JSONDecodeError`.



In my opinion, this is a LiteLLM bug: when handling multiple function calls, it incorrectly concatenates arguments, but hasn’t been fixed yet. For reference, Google ADK added a workaround to defensively handle this issue:



* https://github.com/google/adk-python/pull/759



I think if OpenAI Agents SDK needs to fix this, it may also have to patch the LiteLLM integration layer in a similar way to Google ADK.



That’s what I’ve found so far.
