Just sharing the code snippet working for me so far to workaround this issue:



```python

while not response.output_text:

    # prepare input messages for the next request, adding latest output for LLM context

    input_messages.extend(response.output)



    for output in response.output:

        # only handle known output types

        if output.type not in ["function_call", "reasoning"]:

            continue



        # 'function_call' output type

        if output.type == "function_call":

            func_result = _handle_tool_call(output.name, output.call_id, output.arguments)



            # add the tool call result for the next request

            input_messages.append(

                {

                    "type": "function_call_output",

                    "call_id": output.call_id,

                    "output": str(func_result),

                }

            )



    # TODO: workaroud for OpenAI's bug: https://github.com/openai/openai-python/issues/2561

    # If the last input message is an isolated `reasoning` without a following item, it will raise a 400 code:

    # "Item 'rs_xxxxx' of type 'reasoning' was provided without its required following item."

    last_input_msg = input_messages[len(input_messages) - 1]

    if isinstance(last_input_msg, ResponseReasoningItem) and last_input_msg.type == "reasoning":

        logging.warning(

            "⚠️⚠️⚠️ Removing isolated reasoning message to workaround OpenAI Issue: https://github.com/openai/openai-python/issues/2561"

        )

        input_messages.pop()



    response, estimated_total_cost = _execute_openai_reponses_create_request(

        llm_model,

        llm_reasoning_effort,

        instructions,

        input_messages,

        tool_descriptions,

        temperature=temperature,

    )

```
