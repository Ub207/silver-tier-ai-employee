## Summary



Clarified the OpenAI Agents SDK documentation on guardrails to indicate that they apply to both user input and the agent&#39;s output, preventing misunderstandings about their scope.



## Problem



The existing documentation stated:

&gt; &quot;Guardrails allow you to run checks/validations on user input, in parallel to the agent running. For example, you could screen the user&#39;s input for relevance.&quot;



This wording implied that guardrails are only applicable to **user input**, omitting the fact that they can also run on **agent output**. This could mislead developers into underutilizing guardrails for output validation and filtering.



## Changes Made



Updated the documentation sentence from:



&gt; Guardrails allow you to run checks/validations on user input, in parallel to the agent running. For example, you could screen the user&#39;s input for relevance.



To:



&gt; Guardrails allow you to run checks/validations on user input in parallel to the agent running, and on the agent&#39;s output once it is produced. For example, you could screen the user&#39;s input and agent&#39;s output for relevance.
