> +    async def close(self) -> None:

+        """Close the Redis connection."""

+        await self._redis.aclose()



**[P1] Avoid closing user-supplied Redis clients**



`close()` unconditionally calls `self._redis.aclose()`, yet the class supports injecting an already-managed Redis client. When a caller shares a `Redis` instance across sessions or other components and then calls `session.close()`, this will shut down the connection pool for everyone (`redis.asyncio.Redis.aclose()` renders the client unusable), breaking subsequent Redis operations outside the session. Consider tracking whether the client was created internally or making `close()` a no-op for external clients so shared connections are not inadvertently closed.
