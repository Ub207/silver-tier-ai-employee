jonathanesquivelyolk left a comment (openai/openai-agents-python#1234)



So I use the the dynamic tool-filter option currently. The static tool filtering seems a bit of a pain since I would need a separate server context with those tools defined per each agent. e.g., 4 agents with different tool whitelists = 4 server contexts. 

In general the MCP server handling is a bit of a pain. Two use-cases I would wish for:



1. In-Agent tool filtering given some server(s). You could just pass a whitelist/blacklist of mcp-tools that the agent is allowed to use. You don't need to figure that out by server initialization. e.g., one server context = as many agents with diverse tools from that server as you want



2. Better support for dynamic toolsets. Ex: One agent selects N (most likely relevant) tools from a set of tools available from a server(s) available for the next agent(s) to use. You can do this with dynamic tool filtering and clever uses of the runtime context, but I feel like this could be made to be easier.  



While I understand I can do both of these in the current codebase, it seems a bit non-intuitive and unfriendly for common MCP use-cases. There are many cases where you would prefer to figure out what tools should be used by what agent when you are creating the agent (beyond readability).
