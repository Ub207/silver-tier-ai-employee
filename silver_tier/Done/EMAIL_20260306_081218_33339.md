ruiqiRichard left a comment (openai/openai-agents-python#1943)



Currently I found that gptoss sometimes calls tools that do not exist. I gave it search_online and open_url, but it sometimes calls browse and some other tools that are probably seen during its pretrain. I’m not sure if this is related to the discard of reasoning content or just a model problem or deployment problem.



Other than this, the model can still generate reasonable content, but I’m concerned whether discarding the reasoning would harm the model’s performance in a multiturn agentic task. I saw from the OpenAI cookbook that for better performance in multiturn tool call, it’s recommended to put reasoning back: [](https://cookbook.openai.com/examples/responses_api/reasoning_items)



I know that one way to put reasoning back is to use responses api, but currently no outside api providers like vllm have fully supported Responses api. So would you consider adding a feature that allows the user to put reasoning back when using these third party chat completions APIs? I think this would also make Agents SDK a more general agent framework not exclusive to OpenAI-provided APIs.
