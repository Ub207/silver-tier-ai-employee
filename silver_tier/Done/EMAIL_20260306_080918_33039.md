@sridharvenkat  This is a breaking change in version 0.4.0.



Before 0.4.0: The SDK always converted function outputs to strings automatically.

After 0.4.0: Function calling now supports images and files. Because of this, your output (i.e. dict) is treated as a file output, which causes the error.



Solution:



In your function's final return, convert your function output to a string explicitly::`str({"file_id": "houRsa7pyDBJ3QGke8mM"})`
