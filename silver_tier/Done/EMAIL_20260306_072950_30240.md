DKingAlpha left a comment (openai/openai-agents-python#1716)



You are using python 3.12 where AsyncGenerator only accepts two parameter. Upgrade to python 3.13 or removing type hinting will solve that.

Besides, current `model_settings` might be incompatible with your api endpoint, you can safely remove that too.



Any way, This issue has been resolved, it's caused by following reasons:

1. mcp_servers: ClientSession.list_tools -> await self._write_stream.send `# !!! block`

2. write_stream.send is blocked because read_stream is full.

3. read_stream is full because it runs in main thread, and it is blocked by gradio launch, which use time.sleep(1) loop to block main thread by default





The fix to my example code should be

```patch

--- bug.py      2025-09-12 15:32:02.586745866 +0800

+++ bug.fix.py  2025-09-12 15:31:29.463648135 +0800

@@ -66,7 +66,10 @@

             server_name='127.0.0.1',

             server_port=7860,

             share=False,

+            prevent_thread_lock=True,

         )

+        while True:

+            await asyncio.sleep(1)

```
