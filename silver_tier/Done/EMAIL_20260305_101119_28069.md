surfingdev created an issue (openai/openai-agents-python#1531)



### Describe the bug



Agent produces a duplicated output that fails during parsing.



Output:



```

{

  "content": [

    {

      "messages": [

        { "content": "¡Hi! I'm Susan from XYZ." },

        { "content": "How can I help you?" }

      ],

      "reasoning": "Initial greetings"

    },

    {

      "messages": [

        { "content": "¡Hi! I'm Susan from XYZ." },

        { "content": "How can I help you?" }

      ],

      "reasoning": "Initial greetings"

    }

  ],

  "role": "assistant",

  "reasoning": null,

  "refusal": null,

  "function_call": null,

  "tool_calls": null,

  "audio": null,

  "annotations": null

}

```



Parsing error:



```

  Invalid JSON: trailing characters at line 2 column 1 [type=json_invalid, input_value='{"messages":[{"content":..."}', input_type=str]

    For further information visit https://errors.pydantic.dev/2.11/v/json_invalid

```



### Debug information

- Agents SDK version: 0.2.4

- Python version: 3.12.9

- Model Used: `openrouter openai/gpt-4.1-mini`



### Repro steps



This is a non-deterministic bug. It happens sometimes even with simple pydantic output models such as the example provided.





### Expected behavior



This fails internally at `Runner.run` level. It would be ideal to have some sort of webhooks after LLM generates the output to eventually remove the duplication before the downstream parsing process continues.
