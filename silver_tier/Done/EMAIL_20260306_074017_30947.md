### Body (preview)

**Summary**

In rare cases, Claude Sonnet 4 with extended thinking emits multiple thinking blocks, each with its own signature. Our converter previously kept only the last signature. This change collects all signatures **in order** and joins them with `\n`, enabling proper reconstruction/verification of each block.



**Before**

Input: two thinking blocks with signatures → output stored only the **last** signature.



**After**

Input: two+ thinking blocks with signatures → output stores **all** signatures joined by newline (e.g., `&quot;sig1\nsig2\nsig3&quot;`).



**Implementation**



* Accumulate signatures during parsing instead of overwriting.

* Store the result as a newline-delimited string at `encrypted_content`



**Tests**



* [x] multiple blocks all w/ signatures (order preserved)



**Compatibility / Risk**



* No behavior change for single-block responses.

* Low risk; perf impact negligible.
