NateRVance created an issue (openai/openai-agents-python#1605)



When obtaining tool output from a runner's `result.to_input_list()` and passing it as input to a runner with a harmony-backed model in vLLM v0.10.1.1 with the chat-completions route, the following exception is raised:



`BadRequestError: Error code: 400 - {'error': {'message': 'Tools should have a name! None', 'type': 'BadRequestError', 'param': None, 'code': 400}}` 



This error appears to originate from harmony's [encoding.rs](https://github.com/openai/harmony/blob/508cbaa7f6b0277bd37c9bdf6d4dc8a4d51aada5/src/encoding.rs#L808-L814), which expects that when `role` is `tool`, then a `name` field must be present.



It appears that [chatcmpl_converter](https://github.com/openai/openai-agents-python/blob/a81601a5364de6251bbbb90b654eea9b1475bf98/src/agents/models/chatcmpl_converter.py#L462-L469) is where this functionality should be added.
