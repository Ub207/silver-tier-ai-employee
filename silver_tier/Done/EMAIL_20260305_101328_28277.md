The latest update on this topic:  



We're introducing a simple opt-in option to switch to GPT-5 with best practices (reasoning.effort and verbosity). That said, we'll continue supporting GPT-4.1/GPT-4o as the default model when no specific model is set for an agent. Please check my documentation update draft for more details: https://github.com/openai/openai-agents-python/pull/1535/files 



We believe GPT-5 models offer better performance for many use cases. However, for developers who have been building around GPT-4's behavior, some prompt and setting adjustments may be needed when migrating to GPT-5. Please check out our prompting guide here: https://cookbook.openai.com/examples/gpt-5/gpt-5_prompting_guide



From my observations, a setup like gpt-5-mini + reasoning.effort=minimal + verbosity=low may feel the closest to GPT-4. Even so, there are still differences, particularly around latency and behavior details. Also, minimal reasoning effort does not allow using built-in tools like file search and image generation. Thus, many devs may utilize both gpt-5 and gpt-4.1/4o depending on their specific needs.



We may provide a recommended prompt prefix (similar to what we did with handoffs: https://openai.github.io/openai-agents-python/handoffs/#recommended-prompts) in the future, though that's not decided yet.



We'll keep working on enhancements to improve quality. I hope this brings more clarity, and we really appreciate your understanding.
