### Body (preview)

This addresses an issue that came up while investigating a bug report impacting a wandb/weave user using our agents SDK TracingProcessor



Fixes a race condition where:

1. The streaming response to the query to the agent finishes before the input guardrails finish executing

2. The parent span for both the streaming response and the input guardrails is completed

3. The entire trace gets completed

4. The input guardrails complete and try to trigger on_span_end in the tracing processor for a span that has already completed in a trace that has already completed



It may be the case that this only occurs when there is no next step, since tripwires are awaited there but I&#39;m not sure
