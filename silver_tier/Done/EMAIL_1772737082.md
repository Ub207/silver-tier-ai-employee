Hi, thanks for writing in! I do not think `RedisSession` itself is doing obvious synchronous I/O inside the SDK. Its Redis calls are using `redis.asyncio`, and the actual connection path is lazy and async.



What *does* happen in `run_streamed()` is:



1. `run_streamed()` returns immediately because the background task is created right away.

2. In the background task, the SDK emits an `AgentUpdatedStreamEvent` first.

3. Before any model streaming events (`RawResponsesStreamEvent` / `run_item_stream_event`) can appear, it awaits `prepare_input_with_session(...)`, which in turn awaits `session.get_items()`.
