**<sub><sub>![P0 Badge](https://img.shields.io/badge/P0-red?style=flat)</sub></sub>  Use valid Dapr client arguments when fetching state**



All calls to `DaprClient.get_state` are passing `state_metadata=self._get_read_metadata()`. The Dapr Python client’s API does not accept a `state_metadata` keyword; the parameters are `metadata` and `consistency`. When this code runs against a real Dapr client (unlike the fake test client), Python will raise `TypeError: get_state() got an unexpected keyword argument 'state_metadata'` and the session backend cannot read any data. The same incorrect keyword is used in the other read paths and should be replaced with `metadata` (and the requested consistency should be passed via the dedicated `consistency` or `StateOptions` parameters).
