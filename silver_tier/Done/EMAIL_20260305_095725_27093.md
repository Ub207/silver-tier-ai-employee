When running an Agent, the agent loop automatically decides what to do next until the final output is populated. The `finish_reason` property in Chat Completions API is not exposed as part of the final response because the output data is compatible with Responses API format, which does not return "finish_reason" property. If there is a specific use case you're trying to implement with Agents SDK, sharing more details would be helpful for us to better understand the need.
