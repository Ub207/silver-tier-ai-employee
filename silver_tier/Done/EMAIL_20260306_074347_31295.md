The `Agent` class accepts a `prompt` parameter with incorrect types without raising validation errors.



**Expected Behavior:**

Should raise a `TypeError` or `ValidationError` when `prompt["id"]` is not a string.



**Actual Behavior:**  

Accepts integers, booleans, etc. for the `id` field without error.



**Code to Reproduce:**



```python

from agents import Agent, Runner



# This should raise validation errors but doesn't

agent = Agent(

    name="Test Agent",

    instructions="You are an assistant.",

    model=model,

    prompt={

        "id": 1,           # ❌ Should be str, not int

        "version": 2,      # ❌ Should be str, not int  

        "variables": True, # ❌ Should be dict, not bool

    },

)



print("{agent.prompt}")

```
