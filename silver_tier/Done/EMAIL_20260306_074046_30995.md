Thanks for sharing this and we're sorry for our slow response here. 



Non-OpenAI models often aren't fully compatible with OpenAIChatCompletionsModel even thought those platforms say they're compatible. Our latest recommendation is to use LiteLLM as the adapter for those platforms rather than trying to use OpenAIResponsesModel / OpenAIChatCompletionsModel with different model names and endpoints: https://openai.github.io/openai-agents-python/models/litellm/





Even though LiteLLM does great job for the interoperability, some models may not fully support some features this SDK provides (e.g., Gemini models do not allow tools and structured outputs at a time). That said, for simple use cases, LiteLLM provides seamless integration.



Let us close this issue now, but if you encounter any issues with LiteLLM, please feel free to open a new issue.
