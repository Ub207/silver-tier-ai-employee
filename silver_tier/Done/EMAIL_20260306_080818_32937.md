This issue has been fixed in PR #1927!



## What Was Fixed



The PR changes all Agent parameter validation to raise `UserError` instead of `TypeError`, including:



- ✅ `tools` parameter validation (your exact issue!)

- ✅ `name`, `handoffs`, `instructions`, `model_settings`, `hooks`, etc.



## The Fix



Added proper type checking in `Agent.__post_init__`:



```python

if not isinstance(self.tools, list):

    raise UserError(f"Agent tools must be a list, got {type(self.tools).__name__}")

```



Your exact test case is now included in the test suite:



```python

def test_tools_type_validation_issue_1443(self):

    """Test that UserError is raised when invalid tool type is passed (Issue #1443)"""

    # Original bug: passing a string instead of a list should raise UserError, not AttributeError

    with pytest.raises(UserError, match="Agent tools must be a list, got str"):

        Agent(

            name="TestAgent",

            instructions="Test agent",

            tools="predict_weather",  # type: ignore  # should be a list

        )

```



## Now You Get



```python

UserError: Agent tools must be a list, got str

```



Instead of:

```python

AttributeError: 'str' object has no attribute 'name'

```



Much clearer error message! 🎉



This will be available in the next SDK release. Thank you for reporting this issue with such a clear description!
