# Title



Agent(name=1) accepts int even though name: str is expected



# Body



Describe the bug

The Agent constructor type hints that name should be a str. However, passing an int works without error. This is misleading and could cause unexpected behavior if future code assumes the value is a string.



# Code to Reproduce



from openai import Agent, Runner



agent = Agent(name=1)  # Should raise error, but works

result = Runner.run_sync(agent, "Write a haiku about recursion in programming.")

print(result.final_output)





# Expected Behavior

A TypeError or validation error should be raised if name is not a string.



# Actual Behavior

The code runs fine. Internally, the int seems to be coerced into a string (e.g., "Running agent 1").
